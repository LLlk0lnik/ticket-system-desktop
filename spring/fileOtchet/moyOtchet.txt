Тип приложения и средства разработки
Тип приложения: Десктопное приложение
Используемые технологии:
•	Язык программирования: Java (JDK 8+)
•	Фреймворк: Java Swing (GUI)

Описание сценария прецендентат “Заказ Билетов”
Актор: Пользователь
Цель: Забронировать билет на подходящий рейс
Предусловие: Пользователь авторизован в системе
Основной поток:
1)	Пользователь нажимает на поле «Поиск» на главной странице приложения
2)	Система отображает форму поиска с полями: «Откуда», «Куда», «Дата и время отправления»
3)	Пользователь заполняет все необходимые поля параметрами поездки
4)	Пользователь нажимает кнопку «Найти»
5)	Система выполняет поиск рейсов по заданным параметрам
6)	Проверка: результаты найдены?
a)	Нет → система отображает сообщение «Ничего не найдено» → пользователь изменяет параметры поиска → повторяет поиск (возврат к шагу 3)
b)	Да → проверка: результатов много?
c)	Да → пользователь выставляет фильтры (время отправления/цена/перевозчик/длительность) → система обновляет список результатов согласно фильтрам
d)	Нет → переход к следующему шагу
7)	Пользователь выбирает подходящий рейс из списка доступных вариантов
8)	Пользователь нажимает кнопку «Забронировать» для выбранного рейса
9)	Система отображает форму выбора места с графическим интерфейсом (схема расположения мест в виде интерактивных кнопок)
10)	Пользователь выбирает конкретное место, нажимая на соответствующую кнопку в графическом интерфейсе
11)	Пользователь заполняет форму ввода данных пассажира (поля: ФИО, номер документа, контактные данные)
12)	Пользователь нажимает кнопку «Подтвердить бронирование»
13)	Система обрабатывает заказ, сохраняет данные, отправляет подтверждение и электронный билет
14)	После успешного бронирования система автоматически возвращает пользователя на предыдущую форму интерфейса (к списку доступных рейсов)
Альтернативный поток:
1)	Если выбранное место уже занято, система показывает сообщение и предлагает выбрать другое место
2)	Если при бронировании возникают ошибки в данных пассажира, система отображает соответствующее сообщение и предлагает исправить данные
3)	Если пользователь отменяет процесс бронирования, система возвращает его к просмотру списка рейсов
