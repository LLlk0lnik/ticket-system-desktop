«Кодирование и тестирование приложения»
1. Перечень требований к программному средству
Функциональные требования:
1.	Управление товарами (ведение каталога)
2.	Оформление продаж с поиском товаров
3.	Расчет общей суммы чека
4.	Поддержка разных форм оплаты (наличные/карта)
5.	Формирование и сохранение чеков
6.	Отмена/удаление товаров из чека
7.	Начало новой продажи
Нефункциональные требования:
1.	Производительность: быстрый поиск товаров (<2 сек)
2.	Надежность: сохранение данных при сбоях
3.	Удобство использования: интуитивный интерфейс
4.	Совместимость: работа с Windows
5.	Безопасность: разграничение прав (в перспективе)
 
2. Перечисление выполненных требований
       Полностью выполнены:
1.	Управление товарами
o	Товары загружаются из внешнего файла data/products.txt
o	Формат: Название|Цена|Штрихкод|Категория
o	Автоматическое создание файла при первом запуске
2.	Оформление продаж
o	Поиск товаров по названию (нечеткий поиск)
o	Добавление товаров в чек по клику
o	Автоматический расчет суммы
3.	Расчет суммы
o	Реальное время пересчета при добавлении/удалении
o	Отображение итоговой суммы
4.	Поддержка оплаты
o	Выбор способа: наличные/карта
o	Расчет сдачи при наличной оплате
o	Валидация суммы оплаты
5.	Сохранение чеков
o	Каждый чек сохраняется в отдельный файл
o	Формат: receipts/чек_номер_дата_время.txt
o	Общая база продаж: receipts/продажи_база.txt
6.	Управление чеком
o	Удаление последнего товара
o	Очистка чека (новая продажа)
o	Проверка на пустой чек
7.	Интерфейс
o	Интуитивный дизайн в стиле кассового аппарата
o	Цветовое выделение важных элементов
o	Сообщения об ошибках и успехах
 
3. Перечисление невыполненных или измененных требований
        Не выполнены (временно):
1.	Отчетность за период - нет интерфейса для просмотра отчетов
2.	Работа с кассовой сменой - нет учета смен кассиров
3.	Поддержка сканеров штрихкодов - только ручной ввод
4.	Печать фискальных чеков - сохранение только в файлы
5.	Разграничение прав пользователей - один уровень доступа
       Измененные требования:
1.	Вместо БД SQLite → текстовые файлы (упрощение)
2.	Вместо печати на принтер → сохранение в файлы
3.	Поиск только по названию (без штрихкодов)

6. Инструкция для пользователя
Назначение приложения:
Кассовая система для розничного магазина. Позволяет оформлять продажи, рассчитывать суммы, сохранять чеки.
Возможности:
•	Поиск товаров по названию
•	Оформление продаж
•	Расчет сдачи
•	Сохранение истории продаж
•	Управление ассортиментом через файл

Пошаговая инструкция:
1. Начало работы:
•	Запустите файл CashierInterface.java
•	При первом запуске создастся папка data с файлом товаров
•	Добавьте товары в файл data/products.txt в формате: Название|Цена|Штрихкод|Категория
2. Оформление продажи:
text
Шаг 1: Введите название товара в поле "Поиск товара"
Шаг 2: Нажмите "НАЙТИ" или Enter
Шаг 3: Выберите товар из списка (он добавится в чек автоматически)
Шаг 4: Повторите для всех товаров
3. Оплата:
text
Шаг 1: Выберите способ оплаты (Наличные/Карта)
Шаг 2: Если наличные - введите сумму в поле "Внесено"
Шаг 3: Нажмите "ОПЛАТИТЬ"
Шаг 4: При успехе - чек сохранится в папке receipts
4. Управление:
•	УДАЛИТЬ - удаляет последний товар из чека
•	НОВАЯ ПРОДАЖА - очищает текущий чек и начинает новый
•	При ошибках появляются всплывающие сообщения
Формат файла товаров:
Создайте файл data/products.txt:
text
Название товара|Цена|Штрихкод|Категория
Хлеб Бородинский|45.0|1001|Хлеб
Молоко 3.2%|89.0|1002|Молочные
 
7. Демонстрационный вариант приложения
Сценарий демонстрации:
Дано:
•	Файл товаров с 5 товарами
•	Начальный номер чека: 1
•	Пустая папка receipts
Действия:
text
1. Запуск программы
2. Поиск: "хлеб" → выбор "Хлеб Бородинский"
3. Поиск: "молоко" → выбор "Молоко Простоквашино"
4. Выбор оплаты: "Наличные"
5. Ввод суммы: 200.00
6. Нажатие "ОПЛАТИТЬ"
Результат:
•	Сумма чека: 134.00 ₽
•	Сдача: 66.00 ₽
•	Создан файл: receipts/чек_1_20241215_143025.txt
•	В базе: запись о продаже в receipts/продажи_база.txt
